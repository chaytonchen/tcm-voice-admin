# 中医理疗SaaS平台 - 后台MVP页面设计文档

## 1. 项目概述

本项目是中医理疗SaaS平台的管理后台，主要用于管理租户入驻、门店信息、用户账号等核心业务数据。根据主业务流程，设计MVP（最小可行产品）页面，确保核心业务流程能够顺畅运行。

## 2. 角色定义

| 角色名称 | 角色描述 |
|---------|---------|
| 平台管理员 | 负责平台整体运营，包括租户审核、系统配置等 |
| 租户管理员 | 租户公司的超级管理员，负责创建和管理门店、店长账号等 |
| 销售部审核员 | 负责审核租户入驻申请，创建租户账号 |

## 5. 核心业务流程对应页面

### 5.1 租户入驻流程

1. 租户在小程序端申请入驻 → 无后台页面（小程序端）
2. 销售部审核员登录后台 → 登录页面
3. 进入租户入驻申请列表 → 租户入驻申请列表
4. 查看租户申请详情并审核 → 租户详情审核页
5. 审核通过后，系统自动创建租户ID和后台登录账号 → 租户管理列表

### 5.2 租户创建门店和店长流程

1. 租户管理员登录后台 → 登录页面
2. 进入租户仪表盘 → 租户仪表盘
3. 创建门店基本信息 → 门店创建/编辑页
4. 添加门店店长账号 → 店长账号创建/编辑页
5. 查看门店列表和店长账号列表 → 门店管理列表、店长账号管理

## 6. 技术实现建议

1. **页面布局**：采用左侧菜单 + 顶部导航 + 主内容区的经典后台布局
2. **权限控制**：基于角色的权限控制（RBAC），确保不同角色只能访问对应页面
3. **数据交互**：使用UniCloud云函数处理复杂业务逻辑，简单CRUD操作使用JQL
4. **表单验证**：使用Element Plus的表单验证组件，确保数据完整性
5. **状态管理**：使用Pinia管理全局状态，如用户信息、权限信息等
6. **响应式设计**：确保在不同屏幕尺寸下都能正常显示

## 7. 后续扩展建议
